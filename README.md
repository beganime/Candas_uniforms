# Cadas Uniforms — Django каталог медицинской одежды

Готовый сайт-каталог на Django с админкой Unfold, SQLite, Docker, адаптивным дизайном, загрузкой товаров через админку, заявками и QR-кодами товаров.

## Что внутри

- Страницы: Главная, О компании, Каталог, страница товара.
- Редактируемые тексты, телефоны, адреса, изображения и контакты через админку.
- Товары, категории и дополнительные фотографии через админку.
- Современная админка Unfold.
- Заявки от клиентов без ответа из сайта: видны в админке.
- В заявке сохраняются имя, телефон, email, адрес/город, сообщение, товар, IP-адрес, User-Agent, страница-источник.
- QR-код для каждого товара автоматически генерируется по ссылке товара.
- Сжатие и ресайз изображений при загрузке через Pillow.
- Lazy-loading изображений на страницах.
- Кэширование static/media через Nginx headers.
- Docker + docker-compose.

## Быстрый запуск локально

```bash
cp .env.example .env
docker compose up --build -d
```

Админка:

```text
http://localhost/admin/
```

Логин/пароль берутся из `.env`:

```env
DJANGO_SUPERUSER_USERNAME=admin
DJANGO_SUPERUSER_PASSWORD=change-me
```

После первого входа обязательно поменяйте пароль.

## Запуск на сервере для домена medisinskayaodezhda.ru

1. Укажите DNS A-запись домена на IP сервера.
2. На сервере установите Docker и Docker Compose.
3. Загрузите проект на сервер.
4. Создайте `.env`:

```bash
cp .env.example .env
nano .env
```

5. В `.env` обязательно поменяйте:

```env
SECRET_KEY=длинный_случайный_ключ
DJANGO_SUPERUSER_PASSWORD=сложный_пароль
SITE_DOMAIN=https://medisinskayaodezhda.ru
```

6. Запустите:

```bash
docker compose up --build -d
```

7. Откройте:

```text
http://medisinskayaodezhda.ru/admin/
```

## HTTPS

В проекте указан HTTP Nginx-конфиг. Для HTTPS можно поставить Certbot на сервере или использовать внешний reverse proxy, например Nginx Proxy Manager, Traefik, Cloudflare Tunnel.

## Основные разделы админки

- **Настройки сайта** — название, логотип, телефоны, WhatsApp, email, адрес, домен.
- **Редактируемые страницы** — контент Главной и О компании.
- **Категории** — категории каталога.
- **Товары** — карточки товаров, фото, цены, размеры, цвета, QR-код.
- **Заявки клиентов** — все обращения и заявки по товарам.

## Важное

SQLite подходит для небольшого каталога. Если заявок и товаров станет много, лучше перейти на PostgreSQL.
